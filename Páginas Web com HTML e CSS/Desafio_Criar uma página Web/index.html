O que é HTML

(Voltar)

HTML significa HyperText Markup Language, em português: Linguagem de Marcação de Hipertexto.

É uma linguagem de marcação, não uma linguagem de programação, utilizada na construção de páginas na Web criada por Tim Berners-Lee.

A linguagem de marcação (ou em inglês markup language) são textos delimitados por nomes que o navegador consegue interpretar, sendo que cada nome destes delimitadores tem uma função e comportamento específico.

O HTML existe desde 1991 e atualmente está na versão 5, que veio recheada de recursos e funcionalidades que trazem melhorias para o desenvolvimento web, sua responsabilidade principal é demarcar a estrutura de uma página da web. Essa estrutura do HTML é formada por um conjunto de elementos, ou seja, os hipertextos, que se conectam entre si formando a página. Os elementos HTML ou também chamados de tags HTML, são utilizados para informar ao navegador que tipo de estrutura é essa que está sendo construída, podendo ser títulos, parágrafos, imagens, links, entre outros.

Tim Berners-Lee

Client x Server

(Voltar)

No campo tecnológico, devido a sua amplitude, é importante trazer ao conhecimento alguns termos para melhor compreensão e contextualização do conteúdo à ser estudado.

Neste caso, é abordado sobre o significado de Client e Server para entender em que campo o HTML se encaixa.

Client: Ou cliente no "linguajar" Web, refere-se ao "lado" que consome a informação de algum lugar/dispositivo.

Um exemplo de dipositivo é o notebook, o acesso a essas informações podem ser realizados através de navegadores, também chamados de Browsers.


Server: Ou servidor, é o lado que retorna/fornece o serviço solicitado pelo cliente. Esta comunicação entre cliente e servidor acontece pelo computador e não necessariamente é necessária conexão com a internet.

Alguns tipos de servidores: Proxy (realiza autenticação ou barra o acesso à sites indevidos), Firewall (mais voltado para verificação do acesso externo), E-mail, intranet (servidor web sem internet), DNS ou Domain Name Service.

O servidor pode estar em um computador ou uma rede mais complexa, como é o caso do Facebook que possui vários servidores devido a alta demanda, este local é chamado de Data Center.

Modelo cliente-servidor

Hospedagem de sites - Existem duas opções que podemos escolher ao disponibilizar arquivos e deixa-los acessíveis na internet: criar o próprio servidor ou contratar empresa especializada neste ramo, sendo a última opção a mais indicada.

Ao hospedar o nosso site é criado um domínio ou endereço pelo qual as pessoas conseguem achar o site na internet. Para registro de domínio brasileiro acesse aqui.

As requisições realizadas pelo Client ocorrem em formato HTTP e o servidor retorna em HTML, CSS, Javascript, imagens, texto, dentre outros.

Arquivos estáticos - Arquivos que são enviados ao navegador pelo servidor como HTML, CSS, JavaScript e imagens.

Cache- ferramenta do navegador que possui a função de guardar na máquina do cliente arquivos já acessados para evitar que sejam trafegados novamente.

Design Responsivo ou Responsividade - É uma técnica voltada para a interface de um site, tornando-a de fácil visualização e adaptável a diferentes tipos de navegadores e dispositivos.

Aplicações Web - Soluções criadas que possuem a internet como meio de comunicação entre Client x Server nãos endo necessária a sua instalação, por exemplo, o site do Instagram (www.instagram.com). Aplicativos de celulares não se encaixam nesta classe.

Ferramentas utilizadas

(Voltar)

No curso de Introdução ao HTML realizado neste Bootcamp foi usado como ferramenta o Visual Studio Code, a instalação pode ser realizada por este site.

A extensão instalada no programa foi o Live Server (plugin), abre o arquivo do VSCODE diretamente no navegador e atualiza as alterações automaticamente após salvas no programa. Outra extensão muito útil mas que já vem instalada no programa é o Emmet, ele autocompleta o código tornando a escrita mais dinâmica.

Além do VSCODE existem outros programas de edição de textos que podem ser utilizado para criação de arquivos HTML, o Sublime Text é uma dessas opções.

Geralmente já vem instalado nos computadores, porém, é necessário ter instalado um navegador de sua preferência para rodar os arquivos, neste curso o navegador utilizado é o Google Chrome. No prório navegador é utilizada a Ferramenta do desenvolvedor no navegador (no Google Chrome ela pode ser acessada clicando a tecla F12).

Os arquivos devem ter a extensão .html para ser exibido pelo navegador.

TAGS

(Voltar)

A TAG é um elemento do HTML utilizada para informar ao navegador qual o tipo de estrutura esta sendo construída, pode ser título, parágrafo, imagem, link, etc.

Sua estrutura é formada iniciando com o sinal "menor <", posteriormente o nome e por último o sinal de "maior >". Para fechar a tag é incluído o sinal de barra após o <.

Dentre as tags, existem as de auto fechamento em que não é necessário seguir a orientação de fechar a tag, neste caso é utilizado o sinal de menor no inicio, a barra e sinal de maior ao final.

As tags possuem atributos/propriedades que tornam o arquivo mais dinâmicos e podem ser genéricos ou específicos.
